## Future Usage Rules

- `PageHeader` is rendered by the nearest active route layout when registry metadata exists.
- `PageDocumentMetadata` is rendered by the metadata-owning layout; registered route pages should not add local `<svelte:head>`.
- `AvailabilityBadges` receives `href`; inline availability props are not an authoritative data path.
- `ChildLinkGrid` should eventually receive resolved asset metadata instead of deriving paths from titles.
- `Panel`, `ActionButton`, and `SelectField` should be the default shared controls for settings-like UI.
- `NavTree` owns recursive active/expanded behavior; feed it derived `navigation` data.
